
Описание выполнения команды 2 - "Editing project" в BossMenu:

Процесс Редактирования Описания Проекта:

1. Вывод списка проектов:
При запуске меню редактирования выводится список доступных для редактирования проектов
с использованием метода printAllProjects из класса UserTalkService.

2. Выбор проекта:
Пользователю предлагается ввести ID проекта, который он хочет отредактировать,
через метод inputPositiveInt класса UserInput.

3. Проверка существования проекта:
Используется Optional для безопасной работы с возможно отсутствующим проектом.
Если проект найден, процесс продолжается.

4. Вывод текущего описания проекта:
После выбора проекта на экран выводится его текущее описание, чтобы пользователь мог видеть,
что он собирается изменять.

5. Ввод нового описания:
Пользователь вводит новое описание для проекта с помощью метода inputText.

6. Редактирование описания:
Создается экземпляр ProjectEditService, и вызывается метод editProjectDescription для обновления описания проекта.
Валидация нового описания выполняется внутри сервиса.

7. Отображение результата:
Выводится сообщение с результатом операции (успешное обновление описания или сообщение об ошибке).

Валидация и Обработка Ошибок:
Валидация нового описания проекта выполняется в ProjectEditService с использованием ProjectValidation.
Ошибки валидации формируются в виде списка ErrorDto и возвращаются пользователю, если таковые имеются.
Если проект для редактирования не найден (неверный ID), пользователю выводится соответствующее сообщение.
