
### Общие требования для всех заданий:
- Обеспечение валидации вводимых данных для предотвращения ошибок.
- Разработка должна следовать принципам трехслойной архитектуры,
разделяя приложение на слой доступа к данным, бизнес-логику и
пользовательский интерфейс.
- Важно обеспечить тестирование разработанного функционала для
подтверждения его корректной работы.

### Задание 2: Система управления задачами для командной работы

#### Цели и задачи:
Создайте приложение для управления проектами и задачами внутри
команды. Приложение должно поддерживать регистрацию и
аутентификацию пользователей, создание проектов, добавление
участников к проектам, назначение задач с указанием сроков и
приоритетов, а также комментирование задач и обмен сообщениями
между участниками.
Важной частью проекта является отслеживание прогресса выполнения
задач и проектов.

#### Функционал:
- Регистрация и аутентификация пользователей.
- Управление проектами и задачами, включая создание, назначение и отслеживание.
- Возможность комментирования задач (доп задание: обмен сообщениями между пользователями).
- Отслеживание прогресса по задачам и проектам.


Функционал системы:

Регистрация:
Проверка правильности введенных данных.
Проверка на уникальность логина.
Занесение данных нового пользователя в базу данных после успешной проверки.

Аутентификация:
Проверка наличия пользователя в базе данных по логину и паролю.
Предоставление доступа к функционалу системы в соответствии со статусом пользователя (босс или сотрудник).

Работа с проектами (для босса):
Добавление, редактирование и удаление проектов.
Добавление задач к проектам.
Добавление пользователей к проектам.
Назначение задач пользователям.
Отслеживание прогресса выполнения проекта.

Работа с задачами:
Отслеживание прогресса по задачам.
Изменение статуса и приоритетности задач.
Комментирование задач для обсуждения деталей выполнения.

Обмен сообщениями:
Возможность отправки сообщений между пользователями системы.
Просмотр истории сообщений.

Меню босса и сотрудника:
Босс имеет доступ ко всему функционалу управления проектами и задачами.
Сотрудник может получать список задач, комментировать их,
изменять статус выполнения и обмениваться сообщениями.

Тестирование:
Проведение юнит тестов для проверки корректности работы методов, связанных с управлением пользователями, проектами, задачами и сообщениями.
