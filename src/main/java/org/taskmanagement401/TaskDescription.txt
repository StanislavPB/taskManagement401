
Задание II: Система управления задачами для командной работы


###
Общие требования:
1. Обеспечение валидации вводимых данных для предотвращения ошибок.
2. Разработка должна следовать принципам трехслойной архитектуры, разделяя приложение
на слой доступа к данным, бизнес-логику и пользовательский интерфейс.
3. Важно обеспечить тестирование разработанного функционала для подтверждения его корректной работы.


###
Цели и задачи:

1. Создать приложение для управления проектами и задачами внутри команды.
2. Приложение должно поддерживать:
        - регистрацию и аутентификацию пользователей;
        - создание проектов;
        - добавление участников к проектам;
        - назначение задач с указанием сроков и приоритетов;
        - комментирование задач и обмен сообщениями между участниками;
        - отслеживание прогресса выполнения задач и проектов (важная часть проекта).


####
Функционал:

1. Регистрация пользователей:

    а) проверка правильности введенных данных;
    б) проверка логина на уникальность;
    в) если проверка успешна - внесение данных нового пользователя в базу данных;
    г) присвоение пользователю порядкового (внутреннего) ID.


2. Аутентификация пользователей:

    а) проверка наличия пользователя в базе данных по логину и паролю;
    б) предоставление доступа к функционалу системы в соответствии со статусом пользователя (босс или сотрудник).


3. Меню босса и сотрудника:
   а) босс имеет доступ ко всему функционалу управления проектами и задачами. Он может:
            - создавать\удалять\редактировать проекты;
            - добавлять задачи в проект;
            - редактировать\удалять задачи;
            - добавлять сотрудников в проект;
            - назначать\удалять задачи сотрудникам;
            - определять приоритетность задач;
            - определять дедлайн задачи\проекта;
            - обмениваться сообщениями с сотрудниками всех проектов;
            - отслеживать прогресс выполнения задач и проектов.


   б) сотрудник может:
            - получать список задач;
            - комментировать задачи;
            - изменять статус выполнения задачи;
            - обмениваться сообщениями с другими сотрудниками в рамках проекта;


3. Управление проектами:
            - создание проекта;
            - валидация названия проекта;
            - присвоение проекту порядкового (внутреннего) ID при создании;
            - редактирование проекта;
            - удаление проекта;
            - изменение статуса проекта;
            - добавление задач в проект;
            - удаление задач из проекта;
            - добавление сотрудников в проект;
            - удаление сотрудников из проекта;
            - отслеживание прогресса выполнения проекта.


4. Управление задачами:
            - создание задачи;
            - валидация названия задачи;
            - присвоение задаче порядкового (внутреннего) ID при создании;
            - редактирование задачи;
            - добавление задачи в проект;
            - удаление задачи из проекта;
            - добавление статуса задачи;
            - изменение статуса задачи;
            - добавление приоритета задачи;
            - изменение приоритета задачи;
            - дедлайн задачи;
            - отслеживание прогресса выполнения задачи;
            - добавление сотрудников к задаче;
            - удаление сотрудников из задачи;
            - отслеживание прогресса выполнения задачи;
            - комментирование задачи пользователями.


5. Обмен сообщениями между пользователями (доп. задание):
    а) для сотрудников - возможность отправки и получения сообщений в рамках отдельного проекта;
    б) для босса - возможность отправки и получения сообщений от сотрудников всех проектов;
    в) просмотр истории сообщений.


6. Тестирование:
    а) проведение юнит-тестов для проверки корректности работы методов, связанных с управлением пользователями, проектами, задачами и сообщениями.
